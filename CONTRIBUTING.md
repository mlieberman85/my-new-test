# Contributing to baseline-test

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please be respectful and constructive in all interactions. We are committed to providing a welcoming and inclusive environment.

## How to Contribute

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Use the issue tracker to report bugs
3. Include steps to reproduce, expected behavior, and actual behavior

### Suggesting Features

1. Open an issue describing the feature
2. Explain the use case and benefits
3. Be open to discussion and feedback

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Write or update tests as needed
5. Ensure all tests pass
6. Commit with clear, descriptive messages
7. Push to your fork
8. Open a Pull Request

### Pull Request Requirements

- Clear description of changes
- Reference any related issues
- All tests must pass
- Code should follow existing style conventions

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/my-new-test.git

# Install dependencies
npm install

# Run the project
npm start
```

## Questions?

Feel free to open an issue for any questions about contributing.
